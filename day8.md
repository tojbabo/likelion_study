#### 상속
* 생성자와 private은 상속이 안된다
* 상소기 되면 부모의 멤버(변수, method)를 선언없이 사용 가능

> *모든 클래스는 **object**클래스(최상위 클래스)를 자동으로 **extends**한다*
> cat > pet > object 로 상속하기 때문에 cat도 obj를 상속한다
> 따라서 모든 클래스는 object 클래스를 상속할 수 있다.


구체화: 부모클래스에서 특징을 갖는 자식 클래스를 만드는 것
표준화: 자식클래스들 간의 공통점을 찾아내 부모클래스에 구현하는 것

*항상 자신(자식클래스)보다 부모클래스를 먼저 생성한다*
* 부모클래스 생성자 > 자식클래스 생성자 순으로 호출됨 (* new 자식클래스() 할 경우)
  내부적으로는 자식 생성자에서 부모 생성자를 자동으로 호출하는 것

* super()를 통해서 명시적으로 호출도가능. but, 반드시 첫 라인에서 호출해야함
  + 왜?: 기본적으로 부모의 기본생성자를 호출. 오버로딩 된 생성자가 있다면 super()로 호출해버리면된다.
  + 부모에서 선언된 변수를 부모에서 초기화하기 위해 사용

#### 오버라이딩(overriding)
*부모의 메서드를 자식이 재정의해서 사용*
* 접근 지정자는 같거나 덜 제한 해야 함
* 리턴타입과 파라미터는 동일해야함 > 달라지는 순간 오버로딩(overloading임)

**@Override: 어노테이션**
해당 method가 오버로딩인지 오버라이딩인지, 새로 선언한 것인지 식별하기 어려울때 알아차릴 수 있다

#### super 키워드
*자식에서 부모를 호출할 때 사용*
super.멤버 를 통해 부모의 멤버(변수, 메서드)를 호출할 수 있다

최상위 클래스 - 모든 클래스가 자동으로 상속받는 클래스이면서 가장 먼저 생성되는 클래스
