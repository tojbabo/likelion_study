### 겍체 지향 프로그래밍 (Obejct Oriendted Programming: OOP)
> 현실세계의 객체(속성,동작)라는 개념을 이용해서 프로그래밍
> 대표적인 언어: Java

*객체란 주체가 눈으로 보여지는 모든 사물*

**핵심 구성요소**
1. 속성: 객체의 특성 - 학번, 이름, 전화번호 ...
2. 동작: 객체의 행위 - 등교하기, 공부하기, 하교하기 ...

**객체를 표현하는 방법**
<pre>
객체 -> 클래스
속성 -> 변수
동작 -> 메서드
</pre>

#### 추상화(abstraction)
*필요한 객체 및 속성, 동작을 추출하는 것. (모델링과 비슷한 개념)*



#### main method
main은 Application의 시작점.
main method를 가진 클래스(핸들링 클래스)는 다른 클래스를 핸들링한다.
*main method를 가진 클래스는 독자적으로 실행이 가능하지만, 재사용은 불가능하다.*
모든 Application은 하나의 main method를 갖는다.

#### 생성자
객체가 생성될 떄 호출되는 함수

*오버로딩(OverLoading)으로 여러개의 생성자를 만들 수 있다.*
> **왜 생성자 오버로딩을 하느냐**
> 제한된 데이터만을 가지고 최소한의 정보를 통해 객체를 생성할 수 있게 하기 위해
> 얼굴은 알아도, 이름은 알아도 나이는 모르는 상대방을 만드려고 한다면?
>
> 빈 객체에 하나하나 넣는것 보다는 생성 초기화 속도가 더 빠르다.
